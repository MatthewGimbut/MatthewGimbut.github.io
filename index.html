This involved containerizing several .NET subservices with Docker and rebuilding their DevOps system to accomodate.

I've also contributed to Azure Information Protection, Trusted Launch, and Azure Disk Encryption, in addition to providing 24x7 on-call support for all of the above in the previously mentioned clouds.



Lockheed Martin Space

July 2017 - October 2019

My time at Lockheed Martin has been split between two different projects.

For my first six months at the company, I supported the maintenance of a legacy Command & Control system which was used to plan and schedule contact times between ground antennas and satellites. The software was written in an old version of C# which we eventually upgraded to .NET Core 2.1. It was focused mostly on using phased arrays and parabolic antennas to track and communicate with space vehicles.

After those six months I transitioned into the team which I am still currently on. Our current goal is to build a NOC capable of providing enhanced planning and scheduling capabilities for any type of antenna. In November of 2018, we were able to demonstate our system in use for the Verge project which was presented at Amazon's re:Invent. The system is composed of Java microservies which communicate using REST. Docker containers are built for each service through GitLab's CI/CD pipelines. The services run in an Apache Ignite cluster and persist data in Redis. The NOC can also communicate with other systems though Apache ActiveMQ.

My entire career so far at Lockheed Martin has been in an Agile environment. From January 2018 to April of 2019, I fulfilled the role of Scrum Master for my team of ten people. As a result of Lockheed Martin not having a dedicated position for Scrum Master, my time was split in half between development and team management.



S&P Global Market Intelligence

Summer 2016

I spent the summer interning for S&P Global Market Intelligence in downtown Manhattan. I led development on LogReaderLive, which was a tool used extensively by both the development and quality assurance teams to parse and analyze the logs output by their software. LogReaderLive was a web application that read in several log files at once, each with over 2 million lines of text. It would take just over a second for the tool to parse each file, using regex to break the file up into JSON parseable chunks with associated timestamps. This experience at S&P was also my first time working in a professional Agile environment with teammates located in the US, France, and India.

Message Encoder

Description

This project was a simple message encoder created to allow friends to communicate in secret messages. It takes a string entered by the user, applies a Caesar cipher, and converts it to a binary representation of each character's ASCII value. The 1s and 0s are then replaced by strings entered by the user. It can also be used to decode messages as well. Messages can be send to others over email using the input fields.

C#, Visual Studio

CarTracker

Description

CarTracker served as my senior project in my final year of college. The goal of CarTracker was to provide a sort of guided encyclopedia for car owners who many not be as savvy about the mechanical aspects. The app would take in basic information about the car such as mileage and existing maintenance work to alert the user through email and text when the car was due for service. CarTracker provided the end user with a tool to help organize what work needed to be done on their car and when. Through the use of the Google Maps API, it would also present suggested auto shops to the user based on their location.

Unfortunately, on February 15th of 2018 Edmunds retired their vehicle API program. Without this, CarTracker could no longer function and had to shut down.

The full list of features and specifications can be found here.

Technologies

JavaScript, jQuery, MySQL, Google Maps/Chart API, Edmunds Vehicle API
